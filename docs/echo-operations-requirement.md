## 一、业务背景

### 1.1 角色定位

**Echo（艾可）** 是小九九公司的 AI 新媒体运营，负责：
1. **公众号内容运营**：自动化内容创作、排版、发布
2. **小红书获客运营**：自动采集优质笔记、智能评论互动

### 1.2 业务场景

小九九公司主营跨境电商服务，获客渠道分为：
- **70% 新客户**：来自小红书引流（截流、发笔记、评论互动）
- **30% 老客户**：通过朋友圈和公众号维护

### 1.3 当前痛点

#### 公众号运营痛点
1. **内容创作耗时**：需要人工搜集素材、撰写文章
2. **排版繁琐**：微信编辑器操作复杂，格式调整费时
3. **更新不规律**：人工运营难以保持稳定更新频率
4. **素材分散**：情报中心、知识库、新闻资讯等素材未整合

#### 小红书运营痛点
1. **人工采集效率低**：需要手动浏览、筛选、记录
2. **评论时机难把握**：批量回复容易被平台识别为机器人
3. **优质内容难发现**：缺少自动化的内容质量评估
4. **架构不清晰**：Echo 和 GoldFinger 的职责边界模糊

## 二、核心职能

### 2.1 公众号自动运营

#### 2.1.1 内容创作
- **素材来源**：
  - 飞书多维表格「情报中心」（小东的跨境情报）
  - 知识库中的教程和行业分析
  - 深度搜索能力（实时新闻、政策变化）
- **创作流程**：
  1. Echo 定期（每周 2-3 次）检查情报中心
  2. 筛选适合公众号的素材（标记为"适合公众号"）
  3. 结合知识库教程，生成文章大纲
  4. 撰写完整文章（800-1500 字）
  5. 提交飞书审批流（人工审批）

#### 2.1.2 内容排版
- **技术方案**：
  - **方案 A**：Echo 使用 browser 工具，自动化操作微信编辑器
  - **方案 B**：开发/集成开源排版工具（如 Markdown 转微信格式）
- **排版要求**：
  - 标题格式统一
  - 段落间距合理
  - 图片自动插入
  - 代码块/引用块格式化

#### 2.1.3 审批与发布
- **审批流程**：
  1. Echo 完成文章创作和排版
  2. 提交飞书审批流（通知人工审核）
  3. 人工在飞书中审批（通过/驳回/修改）
  4. 审批通过后，Echo 执行发布（或人工发布）

### 2.2 小红书获客运营

#### 2.2.1 架构设计

**核心原则**：Echo 是决策者，GoldFinger 是执行工具

```
┌─────────────────────────────────────────────────────────┐
│                    Echo（艾可）                          │
│                  AI 新媒体运营                            │
│  - 决策：哪些笔记值得采集                                 │
│  - 决策：哪些评论值得回复                                 │
│  - 决策：回复内容是什么                                   │
└────────────────────┬────────────────────────────────────┘
                     │ 调用 API
                     ↓
┌─────────────────────────────────────────────────────────┐
│                 GoldFinger（金手指）                      │
│              浏览器自动化工具（本地项目）                  │
│  - 执行：打开小红书页面                                   │
│  - 执行：采集笔记内容                                     │
│  - 执行：发送评论                                         │
│  - 执行：模拟人类行为（滚动、停留、点赞）                  │
└─────────────────────────────────────────────────────────┘
```

**职责划分**：

| 角色 | 职责 | 技术实现 |
|------|------|---------|
| **Echo** | 决策层 | OpenClaw Agent，具备上下文理解、学习能力 |
| **GoldFinger** | 执行层 | 浏览器自动化（Playwright/Puppeteer），提供 API |

#### 2.2.2 工作流程

**流程 1：自动采集笔记**

```
1. Echo 读取飞书「现有资源表」
   - 确定目标关键词（如：法国店铺、跨境电商、亚马逊入驻）
   - 确定采集数量和频率
   ↓
2. Echo 调用 GoldFinger API
   - goldfinger_search_notes(keyword, limit)
   ↓
3. GoldFinger 执行
   - 打开小红书搜索页
   - 模拟人类浏览（滚动、停留 2-5 秒）
   - 采集笔记标题、内容、点赞数、评论数
   - 返回结构化数据
   ↓
4. Echo 评估笔记质量
   - 内容相关性（是否匹配业务）
   - 互动质量（点赞数、评论数）
   - 作者活跃度
   ↓
5. Echo 决策：是否立即回复
   - 高质量笔记 → 立即回复
   - 中等质量 → 记录到待跟进列表
   - 低质量 → 忽略
   ↓
6. 如果立即回复：
   - Echo 生成回复内容（自然、有价值、不硬广）
   - 调用 goldfinger_post_comment(note_id, content)
   - GoldFinger 模拟人类操作发送评论
   ↓
7. 记录到飞书多维表格
   - 笔记链接、作者、内容摘要
   - 回复内容、回复时间
   - 后续跟进状态
```

**流程 2：智能评论回复**

**关键约束**：小红书反机器人检测严格，必须像人一样操作

**策略**：
- **即看即回**：浏览笔记时，如果判断值得回复，立即回复（不批量）
- **随机停留**：每条笔记停留 3-8 秒（随机）
- **自然行为**：滚动、点赞、收藏等操作穿插
- **回复间隔**：两次回复间隔 5-15 分钟（随机）
- **内容多样化**：避免重复话术，每次回复都有差异

**GoldFinger API 设计**：
```python
# API 1: 搜索笔记
goldfinger_search_notes(
    keyword: str,
    limit: int = 10,
    sort_by: str = "hot"  # hot/latest
) -> List[Note]

# API 2: 获取笔记详情
goldfinger_get_note_detail(
    note_id: str
) -> NoteDetail

# API 3: 发送评论（模拟人类）
goldfinger_post_comment(
    note_id: str,
    content: str,
    delay_seconds: int = 5  # 随机延迟
) -> CommentResult

# API 4: 模拟人类行为
goldfinger_simulate_human(
    actions: List[str]  # ["scroll", "like", "collect", "stay"]
) -> None
```

#### 2.2.3 学习与优化

**Echo 的成长路径**：
- **初期（实习生）**：
  - 拿不准的笔记，记录到飞书，人工判断
  - 拿不准的回复内容，提交审批
- **中期（熟练工）**：
  - 根据历史数据，自动判断笔记质量
  - 根据回复效果（点赞、回复），优化话术
- **后期（专家）**：
  - 自主决策大部分场景
  - 只有特殊情况才需要人工介入

**记忆系统**：
- 使用 Mem0 记录：
  - 高质量笔记的特征
  - 有效的回复话术
  - 无效的回复话术（避免重复）
  - 用户反馈和人工修正

```

         │                           │
         │ MCP 协议                  │ HTTP API
         ↓                           ↓
┌─────────────────────┐   ┌─────────────────────────────┐
│  Browser 工具        │   │  GoldFinger（本地项目）      │
│  或排版工具插件       │   │  浏览器自动化                │

         │                           │
         ↓                           ↓
```

### 3.2 核心组件

#### 3.2.1 Echo Agent（OpenClaw）

**配置文件**：`instances/kentclaw/data/workspace/echo/AGENTS.md`

**核心能力**：
- 飞书多维表格读写（情报中心、现有资源表）
- 深度搜索能力（web_search）
- 知识库访问（knowledge_search）
- Browser 工具（公众号排版）
- GoldFinger API 调用（小红书运营）
- Mem0 记忆系统（学习优化）

**Heartbeat 配置**：
```json
{
  "heartbeat": {
    "every": "1d",
    "prompt": "检查情报中心，筛选适合公众号的素材；检查现有资源表，决定小红书采集目标。"
  }
}
```

#### 3.2.2 GoldFinger 项目

**技术栈**：
- Node.js + Playwright/Puppeteer
- Express.js（提供 HTTP API）
- 反检测策略（stealth plugin）

**部署方式**：
- 本地运行（与 OpenClaw 同机器）
- 端口：3001（可配置）
- 启动命令：`npm start`

**API 规范**：
- RESTful API
- 返回 JSON 格式
- 错误处理完善
- 日志记录详细

#### 3.2.3 飞书多维表格

**表 1：情报中心（06.1_跨境情报调研）**
- 已有字段：情报内容、来源、时间、风险等级
- 新增字段：
  - `适合公众号`（单选：是/否）
  - `公众号状态`（单选：待创作/已创作/已发布）

**表 2：现有资源表（新建）**
- 字段设计：
  - `资源类型`（单选：店铺/公司/其他）
  - `平台`（单选：亚马逊/eBay/Shopee 等）
  - `国家`（单选：法国/德国/英国等）
  - `数量`（数字）
  - `更新时间`（日期时间）
  - `小红书关键词`（多行文本）

**表 3：小红书笔记库（新建）**
- 字段设计：
  - `笔记链接`（URL）
  - `标题`（文本）
  - `内容摘要`（多行文本）
  - `作者`（文本）
  - `点赞数`（数字）
  - `评论数`（数字）
  - `质量评分`（数字：1-10）
  - `是否已回复`（单选：是/否）
  - `回复内容`（多行文本）
  - `回复时间`（日期时间）
  - `后续跟进`（单选：待跟进/已联系/已成交/已放弃）
