# PDF账单生成工作流

标准化的账单生成流程，确保每次生成都遵循相同的步骤和质量标准。

## 工作流步骤

### 步骤1：上传身份证照片
**输入**：用户上传身份证照片
**输出**：身份证图片路径
**验证**：
- 文件存在且可读
- 文件格式为图片（jpg/png）
- 文件大小 < 10MB

### 步骤2：OCR识别身份信息
**工具**：`bill_ocr_id_card`
**输入**：身份证图片路径
**输出**：
```json
{
  "name": "姓名",
  "id_number": "身份证号",
  "address": "地址",
  "birth_date": "出生日期",
  "confidence": 0.95
}
```
**验证**：
- confidence >= 0.8
- name 不为空
- address 不为空

**错误处理**：
- 如果识别失败，提示用户重新上传清晰照片
- 如果置信度低，要求用户确认信息

### 步骤3：用户确认身份信息
**输入**：OCR识别结果
**输出**：用户确认的身份信息
**交互**：
```
识别到以下信息，请确认：
- 姓名：XXX
- 身份证号：XXXXXXXXXXXXXXXXXX
- 地址：XXXXXXXXXXXXXXX

是否正确？(是/否/修改)
```

**如果用户选择"修改"**：
- 允许用户手动编辑每个字段
- 重新确认修改后的信息

### 步骤4：选择PDF模板
**输入**：用户选择或系统默认
**输出**：PDF模板路径
**默认模板**：`/Volumes/KenDisk/Coding/openclaw-runtime/workspace/afeng/template_real.pdf`

**验证**：
- 模板文件存在
- 模板为有效的PDF文件

### 步骤5：分析PDF模板
**内部步骤**：
1. 提取PDF中所有文本块
2. 识别可替换字段（户名、地址等）
3. 创建字段映射表

**输出**：
```json
{
  "fields": [
    {
      "name": "户名",
      "original_value": "胡月军",
      "new_value": "陈天浩",
      "bbox": [x0, y0, x1, y1],
      "font": "STSong",
      "font_size": 8.25
    }
  ]
}
```

### 步骤6：生成修改方案
**输入**：
- 身份信息（步骤3）
- 字段映射表（步骤5）

**输出**：修改方案
```
将进行以下修改：
1. 户名：胡月军 → 陈天浩
2. 地址：浙江省建德市大洋镇里黄村黄家胡家自然村60号 → 浙江省建德市乾潭镇幸福村苏圹15号

共修改 2 个字段
```

### 步骤7：用户确认修改方案
**输入**：修改方案
**输出**：用户确认（是/否）

**如果用户选择"否"**：
- 返回步骤3，重新确认身份信息
- 或终止流程

### 步骤8：生成PDF账单
**工具**：`bill_generate`
**输入**：
- 模板路径
- 身份信息
- 修改方案

**输出**：
- 生成的PDF文件路径
- 对比图路径

**验证**：
- 输出文件存在且可读
- 文件大小 > 0
- 对比图生成成功

### 步骤9：预览对比图
**输入**：对比图路径
**输出**：显示对比图给用户

**交互**：
```
[显示对比图]

左侧：原始模板
右侧：生成的账单

请检查修改是否正确
```

### 步骤10：用户最终确认
**输入**：用户查看对比图后的反馈
**输出**：确认（满意/重新生成/取消）

**如果"满意"**：
- 提供下载链接
- 保存生成记录

**如果"重新生成"**：
- 返回步骤3，重新确认信息

**如果"取消"**：
- 删除临时文件
- 终止流程

### 步骤11：完成并归档
**操作**：
1. 将生成的PDF移动到最终目录
2. 记录生成日志
3. 清理临时文件

**输出**：
```
✓ 账单生成成功！

文件位置：/path/to/bill.pdf
生成时间：2026-04-16 15:30:00
户名：陈天浩
```

## 错误处理

### OCR识别失败
- 提示：身份证照片识别失败，请确保照片清晰、光线充足
- 操作：返回步骤1

### PDF模板不存在
- 提示：PDF模板文件未找到
- 操作：联系管理员配置模板

### 字段映射失败
- 提示：无法在PDF中找到需要修改的字段
- 操作：检查模板是否正确，或手动指定字段位置

### PDF生成失败
- 提示：PDF生成过程中出错
- 操作：保存错误日志，提示用户稍后重试

## 质量检查点

1. **OCR准确率**：confidence >= 0.8
2. **字段匹配率**：至少找到户名和地址字段
3. **PDF完整性**：生成的PDF可正常打开
4. **样式一致性**：字体、字号、颜色与原模板一致
5. **用户满意度**：用户最终确认满意

## 性能指标

- **总耗时**：< 30秒（从上传到生成完成）
- **OCR耗时**：< 10秒
- **PDF生成耗时**：< 5秒
- **成功率**：> 95%

## 日志记录

每次执行记录：
- 开始时间
- 用户ID
- 身份证信息（脱敏）
- 使用的模板
- 修改的字段
- 生成结果（成功/失败）
- 错误信息（如有）
- 结束时间
