## 二、数值之间的逻辑关系

### 关系1: 电量计算
```
本期电量 = 本期示数 - 上期示数
峰电量 = 10105 - 9343 = 762
平电量 = 2356 - 2215 = 141
谷电量 = 3601 - 3325 = 276
总电量 = 762 + 141 + 276 = 1179 ❌ (模板中显示762，有问题)
```

**发现问题**: 模板中"本期电量762"应该是总电量1179，或者表格中的数据有误。

### 关系2: 电费计算
```
峰段电费 = 276 × 0.5224 = 144.18 ✓
平段电费 = 345 × 0.5224 = 180.23 ✓
谷段电费 = 141 × 0.175 = 24.68 ✓
基金 = 482 × 0.3 = 144.6 ✓
附加 = 100 × 0.1 = 10 ✓
总电费 = 144.18 + 180.23 + 24.68 + 144.6 + 10 = 503.69 ✓
```

### 关系3: 平均电价
```
平均电价 = 总电费 / 总电量
0.661 ≈ 503.69 / 762 = 0.661 ✓
```

## 三、合理变动策略

### 策略A: 保守方案（推荐）
**只变动核心数值，保持电价不变**

变动字段：
1. 本期电量: 600-1200 千瓦时（随机）
2. 上期示数、本期示数（按比例调整）
3. 应缴电费（根据电量和固定电价计算）
4. 平均电价（自动计算）
5. 环比增长（随机 -20% ~ +30%）

不变字段：
- 电价单价（0.5224, 0.175等）
- 基金和附加费率（0.3, 0.1）

**优点**: 
- 逻辑简单，不易出错
- 电价是政府定价，保持不变更真实

### 策略B: 完全方案
**变动所有数值，包括电价**

变动字段：
1. 本期电量: 600-1200 千瓦时
2. 电价单价: 根据地区调整（浙江 vs 四川）
3. 峰谷平比例: 随机调整
4. 基金和附加: 根据总电量比例调整
5. 所有计算字段

**优点**: 
- 更真实，不同地区电价不同
- 数据更丰富

**缺点**: 
- 需要维护各地区电价数据库
- 计算复杂，容易出错

## 四、实施方案（推荐策略A）

### 第1步: 生成随机电量
```python
import random

# 总电量
total_kwh = random.randint(600, 1200)

# 峰谷平分配（参考比例: 峰45%, 平36%, 谷19%）
peak_ratio = random.uniform(0.40, 0.50)
flat_ratio = random.uniform(0.30, 0.40)
valley_ratio = 1 - peak_ratio - flat_ratio

peak_kwh = int(total_kwh * peak_ratio)
flat_kwh = int(total_kwh * flat_ratio)
valley_kwh = total_kwh - peak_kwh - flat_kwh
```

### 第2步: 计算电表示数
```python
# 假设上期总示数
last_total = random.randint(10000, 50000)

# 按比例分配上期示数
last_peak = int(last_total * 0.6)
last_flat = int(last_total * 0.15)
last_valley = int(last_total * 0.25)

# 本期示数 = 上期 + 本期电量
curr_peak = last_peak + peak_kwh
curr_flat = last_flat + flat_kwh
curr_valley = last_valley + valley_kwh
```

### 第3步: 计算电费
```python
# 固定电价（浙江居民电价，可根据地区调整）
PRICE_PEAK = 0.5683  # 浙江峰段
PRICE_FLAT = 0.5683  # 浙江平段
PRICE_VALLEY = 0.2883  # 浙江谷段

# 电费计算
fee_peak = peak_kwh * PRICE_PEAK
fee_flat = flat_kwh * PRICE_FLAT
fee_valley = valley_kwh * PRICE_VALLEY

# 基金和附加（按总电量比例）
fund_kwh = int(total_kwh * 0.63)  # 约63%
fund_fee = fund_kwh * 0.3

addon_kwh = int(total_kwh * 0.13)  # 约13%
addon_fee = addon_kwh * 0.1

# 总电费
total_fee = fee_peak + fee_flat + fee_valley + fund_fee + addon_fee
```

### 第4步: 计算统计数据
```python
# 平均电价
avg_price = total_fee / total_kwh

# 环比增长（随机）
growth_rate = random.uniform(-0.20, 0.30)

# 峰谷分时比例
peak_percent = (peak_kwh / total_kwh) * 100
valley_percent = (valley_kwh / total_kwh) * 100
```

## 七、风险和注意事项

### 风险1: 数值精度
- 电费计算可能有小数点误差
- 需要四舍五入到2位小数

### 风险2: 逻辑一致性
- 必须确保: 总电量 = 峰 + 平 + 谷
- 必须确保: 总电费 = 各项电费之和

### 风险3: 真实性
- 电价必须符合当地政策
- 电量范围要合理（不能太高或太低）

### 风险4: 字段定位
- PDF中有很多数值字段，需要精确定位
- 某些字段可能在表格中，替换时要小心
