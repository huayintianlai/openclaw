## 一、项目背景

### 1.1 业务场景

公司主营跨境电商服务，帮助中国卖家获取欧洲店铺资源。获客渠道主要是：
- 小红书（70%）：通过发笔记、评论区互动引流到微信
- 老客户朋友圈（30%）：复购和转介绍

### 1.2 痛点

1. **小红书获客效率低**：
   - 需要人工浏览大量笔记和评论
   - 识别意向客户耗时
   - 回复话术需要思考
   - 必须"边看边回"（平台反机器人机制）

2. **公众号运营繁琐**：
   - 选题策划需要收集素材
   - 文章撰写耗时
   - 排版工作重复性高

3. **资源利用不充分**：
   - 飞书情报中心（小东跨境雷达）的情报未充分利用
   - 现有店铺资源未系统化管理
   - 获客数据未深度分析

### 1.3 解决方案

创建 Echo（艾可）数字员工，负责：
1. 公众号内容创作和自动化排版
2. 小红书获客任务调度和策略优化
3. 数据分析和学习优化

## 二、核心理念

### 2.1 Echo 的定位

**Echo = 新媒体运营数字员工**

- **不是**：简单的自动化脚本
- **而是**：懂业务、会学习、能决策的 AI Agent

### 2.2 成长路径

```
初期（实习生）：
  - 拿不准的事情 → 人工审批
  - 按照固定策略执行
  - 记录所有操作和结果

中期（熟练工）：
  - 常见场景自己判断
  - 特殊情况人工审批
  - 根据数据优化策略

长期（专家）：
  - 大部分场景自主决策
  - 极少需要人工介入
  - 主动发现机会和风险
```

### 2.3 设计原则

1. **人使用工具**：Echo 是决策大脑，GoldFinger 是执行手脚
2. **数据驱动**：所有决策基于飞书多维表格的数据
3. **可追溯**：所有操作记录到飞书，便于审计和学习
4. **高可用**：工具选择优先考虑稳定性和可维护性

| 角色 | 职责 | 不做什么 |

## 四、核心功能

### 4.1 功能一：动态采集决策

#### 4.1.1 业务流程

```
【每天定时或 Echo 主动触发】
Echo 读取"现有资源表"
  ↓
分析：
  - 哪些平台/关键词最近有新店铺资源？
  - 哪些关键词转化率高？
  - 哪些时间段采集效果好？
  ↓
决策：今天采集 ["cdiscount新店", "速卖通法国", "allegro波兰"]
  ↓
调用 GoldFinger API
  ↓
GoldFinger 执行采集（模拟真人浏览）
  ↓
实时识别意向客户 → 触发飞书审批流
  ↓
人工审批（3 分钟内）
  ↓
审批通过 → GoldFinger 立即回复
  ↓
结果写入"意向客户池"
  ↓
Echo 读取结果 → 分析效果 → 优化下次策略
```

#### 4.1.2 关键点

**小红书反机器人机制**：
- ❌ 批量采集后统一回复（会被识别）
- ✅ 边看边识别、边审批、边回复（像真人）

**实时审批流程**：
```
GoldFinger 识别到意向评论
  ↓
暂停浏览（阻塞等待）
  ↓
发送飞书审批通知
  ↓
人工审批（批准/拒绝/修改话术）
  ↓
GoldFinger 收到结果 → 立即回复
  ↓
继续浏览下一条评论
```

**超时机制**：
- 审批超时时间：180 秒（3 分钟）
- 超时后：跳过该评论，继续浏览

#### 4.1.3 技术方案

**Echo 调用 GoldFinger 的方式（待定）**：

**方案 A：goldfinger CLI**
```bash
# 优点：简单直接
# 缺点：参数传递不够灵活
goldfinger collect "cdiscount新店"
```

**方案 B：HTTP API**
```bash
# 优点：灵活，可传递复杂参数
# 缺点：需要处理 HTTP 请求
curl -X POST http://goldfinger-control:8000/api/task-packages/xhs_echo_customer_acquisition/enqueue \
  -H "Content-Type: application/json" \
  -d '{"config": {"keywords": ["cdiscount新店"], "max_notes": 50}}'
```

**方案 C：飞书表格间接触发**
```python
# 优点：完全解耦，所有操作可追溯
# 缺点：需要 GoldFinger 监听表格变化
feishu_bitable_create_record(
    app_token="xxx",
    table_id="采集任务表",
    fields={"关键词": "cdiscount新店", "状态": "待执行"}
)
```

**推荐**：
- 初期：方案 A（快速验证）
- 中期：方案 B（灵活控制）
- 长期：方案 C（完全解耦）

### 4.2 功能二：公众号运营

#### 4.2.1 业务流程

```
【每周 2-3 次】
Echo 读取"情报中心"（小东跨境雷达）
  ↓
结合"现有资源表"（有哪些平台店铺）
  ↓
结合知识库（教程、历史文章）
  ↓

  ↓
撰写文章（800-1500 字，Markdown 格式）
  ↓
调用排版工具（Markdown → 公众号格式）
  ↓
方式 1：生成 HTML，手动复制到公众号
方式 2：Playwright 自动化粘贴到公众号后台
  ↓
发布到公众号
  ↓
记录到内容日历表格
```

#### 4.2.2 素材来源

1. **飞书情报中心**（小东跨境雷达）
   - `app_token=PthAwTfrViHCr5kS2xUc4gNLnEd`
   - `table_id=tblR0Ox7ZZTEeZay`
   - 内容：平台政策、风险预警、行业动态

2. **现有资源表**（Echo 维护）
   - 当前有哪些平台店铺资源
   - 哪些平台最近有更新

3. **知识库**（OpenClaw knowledge_search）
   - 历史公众号文章
   - 跨境电商教程

4. **小红书热点**（GoldFinger 采集）
   - 高点赞笔记
   - 热门话题

#### 4.2.3 技术方案

**公众号排版工具（待定）**：

**方案 A：Markdown 转公众号（推荐）**
```
优点：
  - 高可用：不依赖公众号后台
  - 效率高：Echo 直接输出 Markdown
  - 开源方案成熟：md2wx、wechat-format

实现：
  1. 部署 md2wx 服务（Docker）
  2. Echo 调用转换 API
  3. 生成带样式的 HTML
  4. 复制到公众号后台（手动或自动化）
```

**方案 B：Playwright 自动化**
```
优点：
  - 完全自动化
  - 可控性强

缺点：
  - 需要维护脚本
  - 公众号改版需要更新
```

**方案 C：开源编辑器 API**
```

  - 稳定

缺点：
  - 需要找到合适的开源项目
```

**推荐**：方案 A（Markdown 转换）+ 方案 B（自动化粘贴）

### 4.3 功能三：策略优化与学习

#### 4.3.1 数据分析

**定期分析（每周）**：
```python
# 1. 读取意向客户池
客户数据 = feishu_bitable_list_records("意向客户池")

# 2. 分析转化率
for 关键词 in 关键词列表:
    采集数量 = count(客户数据, 关键词=关键词)
    审批通过数量 = count(客户数据, 关键词=关键词, 审批状态="已批准")
    转化率 = 审批通过数量 / 采集数量

    # 3. 更新现有资源表
    if 转化率 > 0.3:
        更新优先级为"高"
    elif 转化率 < 0.1:
        更新优先级为"低"

# 4. 记录到 memory
记录优化建议到 MEMORY.md
```

#### 4.3.2 学习机制（未来）

**学习审批偏好**：
```python
# 分析人工审批历史
审批记录 = feishu_bitable_list_records("意向客户池")

已批准案例 = [r for r in 审批记录 if r["审批状态"] == "已批准"]
已拒绝案例 = [r for r in 审批记录 if r["审批状态"] == "已拒绝"]

# 提取特征
批准特征 = 提取特征(已批准案例)
  - 评论长度、关键词、情感倾向
  - 话术风格、回复时机

拒绝特征 = 提取特征(已拒绝案例)

# 训练分类模型（或使用 LLM few-shot）
模型 = 训练模型(批准特征, 拒绝特征)

# 下次识别时，参考模型预测
if 模型.预测(新评论) == "高置信度批准":
    Echo 自主决策，不需要人工审批
else:
    触发人工审批
```

## 六、技术栈

### 6.1 Echo (OpenClaw Agent)

- **平台**：OpenClaw 3.8+
- **模型**：custom/gpt-5.4（主）+ minimax/MiniMax-M2.5（备用）
- **工具**：
  - `feishu_bitable_*`：飞书多维表格操作
  - `knowledge_search`：知识库搜索
  - `memory_*`：记忆系统
  - `exec`：执行命令（调用 GoldFinger）
  - `browser`：浏览器自动化（公众号排版）
  - `tavily_search`：网络搜索

### 6.2 GoldFinger (RPA 引擎)

- **技术栈**：Python 3.10+ + FastAPI + Playwright
- **部署方式**：Docker
- **插件**：`xhs_echo_customer_acquisition`
- **数据库**：PostgreSQL（Neon 云数据库）
- **任务队列**：Redis

### 6.3 飞书集成

- **审批流**：飞书审批 API
- **多维表格**：飞书 Bitable API
- **通知**：飞书机器人

## 九、风险与挑战

### 9.1 技术风险
