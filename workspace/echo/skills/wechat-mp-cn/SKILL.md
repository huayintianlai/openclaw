# 微信公众号监控

文章监控、阅读量追踪、舆情分析。用户让你“看公众号”“总结公众号文章”“搜公众号内容”时，优先使用 `browser` 工具实际打开页面、读取内容、再总结，不要只给泛泛建议。

## 必须执行的默认流程

1. 如果用户给了公众号文章链接：
   - 直接用 `browser` 打开链接
   - 读取标题、公众号名称、发布日期、正文结构、核心观点
2. 如果用户没有给链接，但明确要找某个主题的公众号内容：
   - 先用 `browser` 打开微信搜一搜、搜狗微信搜索或其他可访问页面进行检索
   - 进入最相关的 1-3 篇文章后再读内容
3. 输出时默认包含：
   - 一段简明摘要
   - 3-5 个重点
   - 对跨境业务或内容运营的启发
4. 不要假装看过正文：
   - 如果页面打不开、需要登录、内容被拦截，要明确说明阻塞点
5. 浏览器 profile 选择：
   - 如果当前用户 Chrome 已 attach，可优先复用 `user`
   - 否则默认使用 OpenClaw 的独立浏览器 profile `openclaw`

## 功能

- 📱 文章监控
- 📊 阅读量追踪
- 🔍 舆情分析
- 📢 订阅号/服务号管理

## 使用场景

### 1. 竞品监控

- 追踪竞品发布
- 分析内容策略
- 学习爆款文章

### 2. 舆情监控

- 品牌关键词
- 行业热点
- 危机预警

### 3. 内容分析

- 阅读量趋势
- 点赞/评论分析
- 分享传播

### 4. 文章总结

- 打开公众号文章原文
- 提炼核心观点和论据
- 输出适合老板快速阅读的结论

## 数据字段

| 字段 | 说明 |
|------|------|
| title | 文章标题 |
| author | 作者 |
| publish_time | 发布时间 |
| read_count | 阅读量 |
| like_count | 点赞数 |
| comment_count | 评论数 |

## 注意事项

1. **API 限制**: 需要认证
2. **频率控制**: 避免被封
3. **数据合规**: 仅用于个人研究
4. **优先读原文**: 能打开文章就先读原文，再做总结
5. **优先浏览器**: 这类任务默认先调 `browser`，不是只靠记忆回答
