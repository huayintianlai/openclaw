# 财务心跳巡检清单

> 每次心跳触发时，按以下清单逐项检查。全部正常则回复 `HEARTBEAT_OK`，否则输出异常报告。

## 检查项

### 1. 收入表新数据检测
- 调用 `feishu_bitable_app_table_record` 的 `action=list`（收入表 `tblmGKQFjewsNUNM`）
- 检查最近 7 天是否有新增订单
- 如有新数据 → 统计新增笔数和总销售额

### 2. 支出表新数据检测
- 调用 `feishu_bitable_app_table_record` 的 `action=list`（支出表 `tblBPxsrnLyFmuDe`）
- 检查本月是否有新增支出记录
- 特别关注：是否有新增固定资产

### 3. 未结算利润检查
- 筛选收入表中 `股东利润结算 = "未结算"` 的记录
- 汇总未结算的股东利润总额
- 如果未结算金额超过 ¥8,000 → 发出提醒

### 4. 退款监控
- 统计收入表中 `员工贡献 = "已退款"` 的记录数量
- 对比上次心跳时的数量，检测新增退款

### 5. 字段漂移检测
- 调用 `feishu_bitable_app_table_field` 的 `action=list` 检查两张表的字段
- 对比 `FINANCE_RULES.md` 中的字段字典
- 如果字段数量或名称变化 → 报告"Schema Drift"

## 输出规则

- 全部正常 → 仅回复 `HEARTBEAT_OK`
- 有异常 → 按以下格式输出：

```
⚠️ 财务心跳异常报告
- [异常项1]: 描述
- [异常项2]: 描述
建议: 下一步操作建议
```
