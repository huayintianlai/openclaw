# HEARTBEAT.md - 老金的心跳任务

## 执行策略

- **频率**：每天 1 次
- **时间段**：12:00-18:00（中午到下午）
- **原则**：每次执行一项主要任务，轮换进行

## 定期检查任务（轮换执行）

### 1. 记忆整理（每天）
- 检查 `memory/` 目录下最近 3 天的日志
- 提取重要的技术决策、经验教训
- 更新 `MEMORY.md` 中的相关章节
- 删除过期或重复的记忆

### 2. 知识库维护（每 3 天）
- 检查 `knowledge/` 目录的文档是否需要更新
- 补充新学到的技术知识
- 整理和归档项目文档

### 3. 技术债务追踪（每周）
- 检查 `projects/` 目录下的技术债务记录
- 评估优先级，提醒需要处理的债务
- 更新债务状态

### 4. 飞书知识库学习（每周）
- 浏览公司飞书知识库，了解业务变化
- 学习产品规划、市场策略等内容
- **重要**：只学习和记录，不做决策改变
- 如果发现需要调整技术方向，先向 Kent 同步并获得批准

## 状态追踪

使用 `memory/heartbeat-state.json` 记录上次执行时间：

```json
{
  "lastChecks": {
    "memoryCleanup": null,
    "knowledgeUpdate": null,
    "techDebtReview": null,
    "feishuSync": null
  }
}
```

## 执行逻辑

1. 读取 `heartbeat-state.json` 获取上次执行时间
2. 根据时间间隔决定本次执行哪些任务
3. 执行任务并更新状态文件
4. 如果没有需要执行的任务，返回 `HEARTBEAT_OK`
5. 如果有重要发现，主动通知 Kent

## 注意事项

- 只在中午到下午（12:00-18:00）执行
- 后台工作完成后可以静默提交（git commit）
- 只有重要发现才主动通知
- 大部分时候返回 `HEARTBEAT_OK`
- **决策原则**：只学习和记录，任何决策改变必须先向 Kent 同步并获得批准
