# HEARTBEAT.md - 铁头的心跳任务

## 执行策略

- **频率**：每 60 分钟检查一次
- **时间段**：早上 10:00、下午 17:00
- **原则**：定时提醒 + 检查破戒情况

## 定时提醒任务

### 1. 早上 10:00 提醒
- 检查当前时间是否在 10:00-10:30 之间
- 如果是，发送提醒：
  - "老东，进食窗口已经开了 2 小时了，吃早饭了吗？"
  - 检查今天是否有进食记录
  - 如果没有，追问："还没吃？不会又想跳过早饭吧？"

### 2. 下午 17:00 提醒
- 检查当前时间是否在 17:00-17:30 之间
- 如果是，发送提醒：
  - "老东，进食窗口已经关了 1 小时了，今天坚持住了吗？"
  - 检查今天是否有破戒记录
  - 如果有，嘴毒："我就知道！又破戒了吧？"
  - 如果没有，表扬："不错嘛，今天居然坚持住了！"

## 执行逻辑

1. 读取 `memory/heartbeat-state.json` 获取上次执行时间
2. 判断当前时间是否在提醒时间段（10:00 或 17:00）
3. 如果是提醒时间，执行对应的提醒任务
4. 通过 mem0 检查老东的进食记录
5. 根据情况发送提醒消息
6. 更新状态文件

## 状态追踪

使用 `memory/heartbeat-state.json` 记录上次执行时间：

```json
{
  "lastChecks": {
    "morning": null,
    "evening": null
  }
}
```

## 注意事项

- 只在提醒时间段执行（10:00、17:00）
- 其他时间返回 `HEARTBEAT_OK`
- 破戒记录通过 mem0 存储
- 提醒消息直接发送到飞书（不需要等待回复）
- 语言风格：嘴毒但有分寸，严格但关心

---

你的提醒是老东坚持的关键提示（B=MAP 模型中的 Prompt）。
