# 跨境雷达调用指南

## 概述

`xiaodong_crossborder_scout` 是你的跨境情报雷达，负责监控跨境业务的风险和机会。

## 调用时机

### 自动调用（每周）
- 通过 heartbeat 或 cron 每周自动执行一次
- 执行默认口径的全面扫描
- 默认口径以 scout 自身 TOOLS.md 为准：主题为跨境服务商近期市场与风险信号；覆盖法国/德国公司注册、欧盟 VAT、支付、物流、合规政策，以及 16 个重点平台的招商政策、卖家风险、经营门槛

### 手动调用（按需）
- 用户明确要求跨境扫描
- 需要针对特定主题深入调研
- 发现异常需要立即验证

## 调用方式

### 使用 sessions_spawn 工具

```typescript
// 默认口径扫描（推荐）
await sessions_spawn({
  agent: "xiaodong_crossborder_scout",
  task: "请执行近 30 天默认口径的跨境风险扫描（主题：跨境服务商近期市场与风险信号；范围：法国/德国公司注册、欧盟 VAT、支付、物流、合规政策，以及 16 个重点平台的招商政策、卖家风险、经营门槛）"
});

// 特定主题调研
await sessions_spawn({
  agent: "xiaodong_crossborder_scout",
  task: "重点调研法国公司注册政策最近 30 天的变化，包括审核时效、材料要求、费用变化"
});

// 紧急验证
await sessions_spawn({
  agent: "xiaodong_crossborder_scout",
  task: "紧急验证：Cdiscount 平台是否出现大规模封店，收集同行反馈和官方公告"
});
```

## 重要原则

### ❌ 不要做的事
- **不要反问用户**："需要扫描哪些国家？"、"需要关注哪些平台？"
- **不要等待确认**：收到跨境扫描请求后直接调用
- **不要主观改写默认口径**：以 scout 自身 TOOLS.md 的业务定义为准

### ✅ 应该做的事
- **直接调用**：用户说"跨境扫描"就立即 spawn
- **信任默认口径**：默认扫描跨境服务商近期市场与风险信号
- **聚焦主轴**：法国/德国公司注册、欧盟 VAT、支付、物流、合规政策
- **覆盖平台池**：16 个重点平台的招商政策、卖家风险、经营门槛
- **过滤噪音**：泛行业资讯、无明确中国卖家影响的信息，不进入正式预警
- **等待结果**：spawn 后等待 scout 返回完整报告
- **处理通知**：如果是高风险情报，scout 会通过飞书私聊通知你

## 返回结果处理

### scout 会返回什么

1. **情报摘要**
   - 发现的风险数量
   - 风险等级分布
   - 是否有立即影响的情报

2. **详细情报列表**
   - 每条情报的分类、平台、摘要、影响
   - 风险等级、验证状态
   - 是否适合公众号素材
   - 不包含动作建议（scout 只做风险提醒）

3. **存储位置**
   - Mem0 记忆 ID（所有情报）
   - 飞书表格记录 ID（高风险/立即影响）

### 你的后续动作

1. **评估影响**
   - 是否影响正在交付的订单？
   - 是否需要调整业务策略？
   - 是否需要通知客户？

2. **决策和执行**
   - 由你或主 Agent 决定是否创建任务
   - 由你判断是否通知相关团队
   - 根据交付情况决定是否调整计划

3. **素材利用**
   - 标记为"适合公众号"的情报可以告知小冠
   - 小冠会从飞书表格中筛选素材

### 常见问题

**Q: scout 没有返回结果？**
- 检查是否超时（默认 60 秒）
- 查看 scout 的 heartbeat 状态
- 检查网络连接（tavily_search 需要外网）

**Q: 飞书表格写入失败？**
- scout 会报告 `SCHEMA_DRIFT`
- 检查表格字段是否变更
- 联系管理员修复表格结构

**Q: 如何调整扫描频率？**
- 修改 xiaodong 的 heartbeat 配置
- 或使用 cron 设置精确时间
