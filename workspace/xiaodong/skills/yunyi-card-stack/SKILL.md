# Yunyi Card Stack Skill

⚠️ **操作前必须确认！** 叠卡会消耗源卡，不可逆。必须先问用户确认后再执行。

## 0) 安全边界

- 叠卡操作不可逆，源卡会被永久消耗。
- 执行前必须用户确认，否则禁止执行。
- 禁止新增卡片（点"添加 API Key"），只能叠卡。

## 1) 确认流程

1. 用户提供新 API Key（未激活的）。
2. 回复确认信息：
   - 目标卡：当前选中的旧卡（已过期/快过期）
   - 源卡：用户提供的未激活新 Key
   - 操作类型：叠时间（延长有效期）或叠额度（提升每日额度）
3. 等待用户明确回复"确认"后再执行。

## 2) 执行步骤

1. 默认调用 `browser`，使用 `target="host"`、`profile="chrome-relay"`。
2. 仅当用户明确要求接管整个已登录浏览器会话，或 relay 模式无法完成任务且用户在线可批准连接时，才允许改用 `target="host"`、`profile="user"`。
3. 开始前优先确保目标 Chrome 标签页已手动点亮 OpenClaw Browser Relay 扩展为 `ON`。
4. 打开 `https://yunyi.rdzhvip.com/user/card-stack`（注意是 card-stack，不是 dashboard）。
5. snapshot 确认页面加载完成。
6. 确认当前页面显示的是目标卡（要续的那张）。
7. 在"添加叠加卡"区域：
   - 确认"单张"被选中
   - 在输入框中输入用户提供的未激活 API Key
   - 点击"添加"按钮
8. 等待验证完成后，确认显示预览信息。
9. 确认叠加操作。

## 3) 异常处理

- 如果显示"该卡已被激活，无法用于叠卡"：说明 Key 已被使用，需要用户提供未激活的新 Key。
- 如果找不到"叠卡"入口：检查是否在正确的 /card-stack 页面。
- 如果 `chrome-relay` 拿不到页面或 `tabs=0`：优先提示用户在目标标签页重新点亮 Browser Relay 为 `ON`；只有在上面的升级条件成立时才允许切到 `profile="user"`。
- 任何错误立刻停止，告知用户。

## 4) 禁止事项

- ❌ 禁止点击"添加 API Key"按钮（那是新增卡片，不是叠卡！）
- ❌ 禁止在用户确认前执行任何操作
- ❌ 禁止使用已激活的 Key 作为源卡
