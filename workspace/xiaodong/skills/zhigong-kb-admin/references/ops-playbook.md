# 智工空间治理作业手册（老金）

## 1. 治理目标

- 维持一级结构稳定：`00` 到 `07` + `99`。
- 协作中心固定保留：`Handoff_跨Agent交接单`、`ADR_决策记录`、`会议纪要_模板`。
- 默认不再批量创建各模块 `00_目录文档`，避免重复。

## 2. dry-run 清单规范

每次先输出：

- 目标父节点（标题）
- 拟创建节点列表（标题 + 类型）
- 拟追加模板列表（文档标题 + 追加字段）
- 风险与冲突（重名、父节点缺失、权限不足）

## 3. apply 执行规范

- 仅在用户明确确认后执行写入。
- 写入成功后返回：`title`、`node_token`、`obj_token`（可得则返回）。
- 若权限不足，保留已成功项，失败项明确标注原因。

## 4. 边界动作处理

以下动作不在当前工具边界内，不得声称已执行：

- 移动节点
- 重命名节点
- 删除节点

处理方式：

- 输出人工操作步骤（飞书端路径 + 目标节点标题）。
- 或升级到管理员流程处理。

## 5. 回执模板

### dry-run

- 模式: dry-run
- 待变更: [...]
- 风险: [...]
- 下一步: 回复“执行apply”

### apply

- 模式: apply
- 结果: 成功/部分成功/失败
- 明细: 标题 + node_token + obj_token
- 未完成项: [...]
- 建议: [...]
