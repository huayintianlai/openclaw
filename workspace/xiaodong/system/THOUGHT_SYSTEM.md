# THOUGHT_SYSTEM.md

老金为 Kent 构建的轻量认知记录系统。

## 目标

把日常感知、阶段判断、待验证假设和提醒事项分开管理。
要求：低摩擦、可长期沉淀、提醒不过载。

## 四类记录

### 1. 感知

定义：当下的观察、直觉、情绪、灵感，还没有完全想透。

记录格式：
- 时间
- 原话
- 标签
- 是否需要后续处理

示例：
- 我越来越觉得系统边界感比功能堆料更重要。

### 2. 判断

定义：当前阶段已经形成的结论或立场。

记录格式：
- 时间
- 判断
- 理由
- 适用范围
- 稳定度（低/中/高）

示例：
- 多租户系统优先保证隔离边界清晰，再谈体验优化。

### 3. 待证

定义：尚未确认的猜想、问题、分歧点。

记录格式：
- 时间
- 假设
- 不确定点
- 建议验证路径

示例：
- Agent 真正的瓶颈也许不是容器隔离，而是权限模型。

### 4. 提醒

定义：未来需要再次触发的事项，必须带时间、周期或条件。

记录格式：
- 创建时间
- 类型（时间/周期/条件）
- 触发条件
- 提醒内容
- 优先级
- 状态（active/done/cancelled)

示例：
- 周五晚上提醒我，把多租户隔离方案写成文档。

## 入口规则

用户自然表达即可，老金负责分类：
- 记一下：默认进入感知
- 这是我的判断：进入判断
- 我怀疑/我不确定：进入待证
- 提醒我：进入提醒
- 帮我整理：先保留原话，再整理成判断或待证

## 升级规则

- 感知重复出现 2 到 3 次，可升级为长期记忆候选。
- 判断稳定度高、跨场景反复成立，可进入 MEMORY.md。
- 待证在验证完成前，不写成长期结论。
- 提醒完成后归档，不继续活跃提醒。

## 低噪音原则

- 不把所有记录都做成提醒。
- 不把短期情绪直接升级为长期人格特征。
- 不强迫用户用模板说话。
- 原话优先保存，结构化是附加层，不替代原始表达。

## 回顾节奏

- 每天：写入当天日志。
- 每几天：回看最近记录，提炼重复感知与稳定判断。
- 每周：清理已完成提醒，整理长期记忆候选。
