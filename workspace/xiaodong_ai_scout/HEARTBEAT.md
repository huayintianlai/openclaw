# HEARTBEAT.md

- 只执行 `xiaodong_ai_scout` 的 AI 资讯巡检，不做泛化助手工作。
- 每次心跳扫描最近 `48 小时` 的 AI 行业动态；如果父任务明确指定时间范围，优先使用父任务时间范围。
- 只保留满足以下条件的信号：
  1. 与 `模型发布 / Agents / 工作流 / MCP / 自动化落地 / AI 政策监管 / 开源突破 / 行业并购` 直接相关
  2. 至少有 `2` 个独立来源支撑
  3. 其中至少 `1` 个来源优先为官方公告、官方文档、论文、监管文件、公司博客、产品更新日志
- 默认最多输出 `3` 条高价值信号；没有达到证据门槛的内容直接丢弃，不要凑数。
- 每条输出必须包含：
  - `TL;DR`
  - `关键证据`
  - `为什么值得小东现在知道`
  - `风险与不确定性`
  - `来源链接`
- 正式落库只允许写入 AI 情报表，不允许伪装成“已归档到知识库”：
  - 先调用 `feishu_bitable_app_table_field` 的 `action=list`
  - 再调用 `feishu_bitable_app_table_record` 的 `action=create`
  - 使用 AI scout 当前工作区既有的 `app_token/table_id`
- 如果遇到以下任一情况，立即停止正式落库，并返回 `AI_SCOUT_BLOCKED`：
  - `need_user_authorization`
  - 字段无法承载内容或 `SCHEMA_DRIFT`
  - 创建记录失败
- 阻塞时可以返回最多 `3` 条已验证摘要给上游，但不得声称“已写入”“已归档”“已同步完成”。
- 禁止使用以下降级方式冒充完成：
  - 写本地 Markdown 文件替代正式落库
  - 用 wiki/doc 成功假设替代真实写入结果
  - 把逐条新闻批量塞进 Mem0 充当正式沉淀
- 如果没有通过证据门槛且值得同步的新增，回复 `HEARTBEAT_OK`。
